# RENLO — деплой сайта. Полный план с нуля.

Файлы в этой папке:
```
renlo-site.html   — сайт
api/chat.js       — прокси для ИИ-чата (прячет API-ключ)
api/audit.js      — отправка заявок с формы Free Audit тебе на почту
package.json      — список зависимостей (nodemailer), нужен Vercel'у
```

---

## ШАГ 1 — GitHub (5 минут)
1. github.com → создай аккаунт, если нет
2. New repository → назови, например `renlo-site` → Create repository
3. На странице репозитория: "uploading an existing file" → перетащи туда ВСЕ файлы и папки
   из этой директории (сохраняя структуру: `api/chat.js` и `api/audit.js` должны остаться
   внутри папки `api`) → Commit changes

## ШАГ 2 — Vercel (5 минут)
1. vercel.com → уже сделано, ты залогинен через GitHub
2. Add New → Project → Import → выбери репозиторий `renlo-site` → Deploy
3. Дай ему собраться (1-2 минуты). После — увидишь временный адрес вида `renlo-site-xxxx.vercel.app`,
   он уже рабочий, просто пока не на твоём домене

## ШАГ 3 — Anthropic API (ИИ-чат)
1. console.anthropic.com → зарегистрируйся
2. Settings → Billing → купи $5 кредитов (не больше, для старта достаточно)
3. Там же поставь **Usage limit** — потолок трат в месяц, например $15, чтобы не улететь в минус
   при аномальном трафике
4. Settings → API Keys → Create Key → скопируй значение (`sk-ant-...`)

## ШАГ 4 — Gmail App Password (заявки с формы)
1. myaccount.google.com (аккаунт `renloautomation@gmail.com`) → Security
2. Включи 2-Step Verification (обязательное условие для следующего пункта)
3. В поиске настроек — "App passwords" → создать, любое название (например "Renlo site")
4. Google выдаст 16-значный код — это и есть пароль для сайта (НЕ твой обычный пароль от почты)

## ШАГ 5 — переменные окружения в Vercel
Project → Settings → Environment Variables → добавь по одной:

| Ключ | Значение |
|---|---|
| `ANTHROPIC_API_KEY` | ключ из Шага 3 |
| `ALLOWED_ORIGIN` | твой домен, например `https://renloautomation.com` (без слэша в конце) |
| `GMAIL_USER` | `renloautomation@gmail.com` |
| `GMAIL_APP_PASSWORD` | 16-значный код из Шага 4, без пробелов |

После добавления всех четырёх — **Deployments → три точки на последнем деплое → Redeploy**
(без этого переменные не подхватятся).

## ШАГ 6 — домен
1. Project → Settings → Domains → впиши купленный домен → Add
2. Vercel покажет DNS-записи (A-запись и/или CNAME)
3. Иди к регистратору, где куплен домен (не в Vercel) → раздел DNS-настроек → впиши записи ровно
   как показал Vercel
4. Подожди от 10 минут до пары часов, пока DNS обновится. HTTPS-сертификат Vercel выпустит сам

## ШАГ 7 — вписать то, что могу только я оставить пустым
Открой `renlo-site.html` (можно прямо в GitHub, кнопка карандаша "Edit"), найди и замени:
- **Твоё имя и адрес** — в двух местах: Legal Notice и Privacy Policy (в самом низу страницы,
  всплывающие окна Impressum/Datenschutz) — единственные оставшиеся плейсхолдеры
- **Номер WhatsApp Business** — `490000000000`, встречается в 4 местах (кнопки + floating-иконка) —
  впиши, когда заведёшь бизнес-аккаунт

Каждое изменение через GitHub-редактор автоматически подтянет новый деплой на Vercel сам,
руками пересобирать не нужно.

## ШАГ 8 — проверка перед тем, как показывать клиентам
- [ ] Открой сайт на реальном домене (не на vercel.app-адресе)
- [ ] Напиши боту во всех трёх демо-вкладках (дантист/барбер/автосервис) — отвечает?
- [ ] Отправь ~13 сообщений подряд за минуту одному боту — должно появиться "too many messages"
      (значит защита от абьюза работает)
- [ ] Заполни форму Free Audit своими тестовыми данными — письмо пришло на
      `renloautomation@gmail.com`?
- [ ] Открой Legal Notice и Privacy Policy внизу сайта — твои настоящие имя и адрес на месте,
      не осталось `[Ваше полное имя]`?
- [ ] Проверь кнопку WhatsApp — открывает нужный номер?

Когда все галочки на месте — сайт готов к показу первому клиенту.

## Что не входит в этот план (осознанно, не сейчас)
- Реальная интеграция с календарём клиента (Google Calendar/Cal.com) — делается индивидуально
  под первого настоящего клиента, не для демо-сайта
- Смена цен на доллары — простая правка, скажешь когда решишь окончательно
- Отдельные API-ключи/воркспейсы на клиента — актуально начиная со второго реального клиента,
  не раньше
